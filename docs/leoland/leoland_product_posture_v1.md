# Leoland Product Posture v1

## Current posture

Leoland is currently:

- terminal-first
- prototype-scale
- single-country
- single-turn
- locally runnable on stackhost1

It is not yet:

- a web app
- a deployable public product
- a multi-system simulation
- a polished game shell

## Best next runtime step

The next runtime should be:

- a clean first playable shell
- still lightweight
- still local-first
- still understandable to Leo

The best immediate candidate is a small local UI or structured terminal flow over the current TypeScript engine, not a full hosted web product.

## Eventual public posture

If `leoland.jon-trading.com` exists later, it should probably host one of these:

- a playable prototype
- a world viewer
- a simulation dashboard

It should not start life as a broad product platform before the core world-state and turn loop are real.

## Why this matters

This keeps Leoland grounded in:

- worldbuilding first
- values first
- runtime clarity
- easy iteration

rather than pushing it into premature app architecture.
